# AI 视觉对话助手 — 设计文档

## 1. 项目概述

本应用是一款**端云协同**的多模态对话助手：用户在浏览器中开启摄像头与麦克风，通过语音（或文字）与 AI 交流；AI 结合**当前摄像头画面**与**用户话语**给出恰当的中文回复，并可通过浏览器语音合成播报。

---

## 2. 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                     端侧（浏览器 Edge）                        │
│  ┌──────────┐  ┌──────────────┐  ┌──────────────────────┐  │
│  │ 摄像头预览 │  │ Web Speech   │  │ 帧采集 & 压缩         │  │
│  │ (本地渲染) │  │ API 语音识别  │  │ (按需、降采样、去重)   │  │
│  └──────────┘  └──────────────┘  └──────────────────────┘  │
│  ┌──────────┐  ┌──────────────┐                              │
│  │ Web Speech│  │ 对话 UI      │                              │
│  │ API 播报  │  │ & 状态监控   │                              │
│  └──────────┘  └──────────────┘                              │
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTPS / JSON（仅用户说话结束时）
                           ▼
┌─────────────────────────────────────────────────────────────┐
│                   云端代理（Node.js Server）                  │
│  API Key 托管 · 请求限流 · 对话历史截断 · 调用多模态 LLM      │
└──────────────────────────┬──────────────────────────────────┘
                           │
                           ▼
┌─────────────────────────────────────────────────────────────┐
│              多模态大模型 API（OpenAI 兼容接口）               │
│  视觉理解 + 语言生成（如 gpt-4o-mini / 通义千问 VL 等）        │
└─────────────────────────────────────────────────────────────┘
```

### 数据流

1. 用户点击「开始对话」→ 浏览器申请摄像头/麦克风权限
2. 端侧启动 Web Speech API 连续语音识别（`zh-CN`）
3. 用户说完一句话 → 触发 `onFinalTranscript`
4. 端侧从 `<video>` 抓取**一帧**，压缩至 ≤512px 宽、JPEG 0.65 质量
5. 与上一帧做简易场景变化检测 → 无变化则**跳过图片上传**（纯文本请求）
6. 后端代理转发至多模态 API，`image detail: low` 降低视觉 token 成本
7. AI 回复返回前端 → 展示在对话区 + Web Speech API 语音播报

## 3. 用户故事：计划 vs 实现

| # | 用户故事 | 计划 | 最终实现 | 说明 |
|---|---------|------|---------|------|
| US-1 | 作为用户，我希望看到实时摄像头画面，确认 AI「能看到」什么 | ✅ | ✅ | 本地 `<video>` 预览，不上传视频流 |
| US-2 | 作为用户，我希望用自然语音与 AI 对话，无需打字 | ✅ | ✅ | Web Speech API 连续识别，显示实时中间结果 |
| US-3 | 作为用户，我希望 AI 结合画面内容回答我的问题 | ✅ | ✅ | 说话结束时抓拍帧，多模态 API 图文理解 |
| US-4 | 作为用户，我希望听到 AI 的语音回复，交互更自然 | ✅ | ✅ | Web Speech API TTS，可开关 |
| US-5 | 作为用户，我希望查看完整文字对话记录 | ✅ | ✅ | 聊天气泡 UI，区分用户/AI |
| US-6 | 作为用户，语音识别不可用时仍能输入文字 | ✅ | ✅ | 底部文本输入框 + 回车发送 |
| US-7 | 作为用户，我希望了解每次请求的资源消耗 | ✅ | ✅ | 状态栏显示 tokens、请求次数、是否含画面 |
| US-8 | 作为用户，我希望 AI 记住最近几轮对话上下文 | ✅ | ✅ | 保留最近 6 轮历史发给模型 |
| US-9 | 作为用户，我希望支持「纯语音唤醒、免按钮」持续对话 | ✅ | ❌ | 受浏览器 Speech API 限制，需手动点开始；未实现 VAD 唤醒词 |
| US-10 | 作为用户，我希望在弱网环境下仍能流畅使用 | ✅ | ⚠️ 部分 | 端侧 STT/TTS 可离线；LLM 仍需网络 |
| US-11 | 作为用户，我希望切换前后摄像头 | ✅ | ❌ | 时间有限，仅默认前置摄像头 |
| US-12 | 作为用户，我希望云端 STT 作为 Web Speech 的备选 | ✅ | ❌ | 成本较高，未实现；文档中保留扩展方案 |

**实现率：8/12 完整实现，1 项部分实现，3 项未实现（均在文档中说明原因）。**

## 4. 运营成本控制：技巧 vs 实际采用

| 技巧 | 描述 | 是否采用 | 实现位置 |
|------|------|---------|---------|
| **本地音视频预览** | 视频流仅在浏览器渲染，不上传 | ✅ 采用 | `useMediaStream.ts` |
| **端侧语音识别 (STT)** | 使用 Web Speech API，免云端 ASR 费用 | ✅ 采用 | `useSpeechRecognition.ts` |
| **端侧语音合成 (TTS)** | 使用 Web Speech API，免云端 TTS 费用 | ✅ 采用 | `useSpeechSynthesis.ts` |
| **按需抓拍，非持续推流** | 仅在用户说完话时抓一帧，非 30fps 上传 | ✅ 采用 | `App.tsx` + `frameCapture.ts` |
| **图像降采样压缩** | 缩至 ≤512px、JPEG 质量 0.65 | ✅ 采用 | `frameCapture.ts` |
| **场景变化检测** | 画面未变时跳过图片，发纯文本请求 | ✅ 采用 | `isSceneChanged()` |
| **低精度视觉模式** | API 请求 `detail: low` 降低视觉 token | ✅ 采用 | `server/routes/chat.ts` |
| **对话历史截断** | 仅保留最近 6 条消息上下文 | ✅ 采用 | `chat.ts` |
| **回复长度限制** | `max_tokens: 300` 控制输出成本 | ✅ 采用 | `chat.ts` |
| **服务端请求限流** | 每 IP 每分钟最多 10 次请求 | ✅ 采用 | `server/index.ts` |
| **Token 用量可视化** | 前端展示累计 tokens，便于感知成本 | ✅ 采用 | `StatusBar.tsx` |
| **视频流云端分析** | 持续上传视频到云端做实时理解 | ❌ 未采用 | 成本过高 |
| **云端 STT/TTS** | 使用 Azure/OpenAI 语音服务 | ❌ 未采用 | 端侧方案已满足需求且免费 |
| **本地小模型推理** | 端侧部署轻量 VLM | ❌ 未采用 | 浏览器算力与模型体积限制 |
| **CDN 缓存图片** | 七牛云 CDN 缓存重复帧 | ❌ 未采用 | 当前按需单帧，重复率低；可作为扩展 |
| **语义缓存** | 相似问题直接返回缓存答案 | ❌ 未采用 | MVP 阶段未实现 |

### 成本估算（单次对话）

假设使用 `gpt-4o-mini` + `detail: low` 图片：

| 组件 | 费用来源 | 估算 |
|------|---------|------|
| STT | 端侧 Web Speech | **$0** |
| 视觉理解 | ~85 tokens (low detail) | ~$0.0001 |
| 文本输入/输出 | ~200 tokens | ~$0.0001 |
| TTS | 端侧 Web Speech | **$0** |
| **合计** | | **~$0.0002/轮** |

对比「持续视频流 + 云端 ASR/TTS」方案，预计节省 **90%+** API 费用。

## 5. 技术选型

| 层级 | 技术 | 理由 |
|------|------|------|
| 前端 | React + Vite + TypeScript | 开发效率高，生态成熟 |
| 端侧语音 | Web Speech API | 零成本、低延迟、浏览器原生 |
| 后端 | Express + TypeScript | 轻量代理，保护 API Key |
| 多模态 AI | OpenAI 兼容接口 | 可切换 OpenAI / 通义 / DeepSeek 等 |
| 部署 | 前后端分离，支持任意 Node 主机 | 灵活 |

---

## 6. 安全与隐私

- API Key 仅存服务端 `.env`，不暴露给前端
- 摄像头画面**不持久化、不录制**，仅说话时抓拍单帧
- 服务端限流防止滥用
- 建议生产环境使用 HTTPS
